# Code of Conduct

## Summary

Be respectful. Be constructive. Remember that the people using this project may interact with technology differently than you do.

## Expected behavior

- Treat all contributors and users with respect regardless of disability status, assistive technology used, experience level, identity, or background.
- Use clear, direct language. Avoid jargon when simpler words work.
- When discussing accessibility, defer to the experience of people who use assistive technology. If someone says the output is confusing with their screen reader, that is a valid bug report, not a matter of opinion.
- Provide constructive feedback on contributions. Explain what needs to change and why.
- Respect that people interact with code and documentation in different ways. What is obvious visually may not be obvious auditorily, and vice versa.

## Unacceptable behavior

- Dismissing accessibility concerns as edge cases or low priority.
- Questioning whether someone "really" has a disability or needs accommodations.
- Harassment, personal attacks, or discriminatory language of any kind.
- Publishing private information about others without their consent.
- Trolling, deliberate provocation, or sustained negativity.

## Scope

This code of conduct applies to all project spaces: issues, pull requests, discussions, and any other communication channel associated with this repository.

## Reporting

If someone violates this code of conduct, contact the maintainer directly at the email address listed in the repository owner's GitHub profile. Reports will be reviewed and addressed. You will not be retaliated against for filing a report in good faith.

## Enforcement

Responses to violations may include a private warning, a public warning, temporary ban, or permanent ban from the project, depending on severity and pattern.
